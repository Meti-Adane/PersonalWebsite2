How rem Units Translate to Pixel Values

When using rem units, the pixel size they translate to depends on the font size of the root element of the page, i.e. the html element. That root font size is multiplied by whatever number you’re using with your rem unit.
For instance, with a root element font size of 16px, 10rem would equate to 160px, i.e. 10 x 16 = 160.

How em Units Translate to Pixel Values

When using em units, the pixel value you end up with is a multiplication of the font size on the element being styled.
For example, if a div has a font size of 18px, 10em would equate to 180px, i.e. 10 x 18 = 180.
The font size, for many browsers, by default is 16px. Unless overridden meaning the style sheet has no font size specified then it is determined by the browser and the standard default font size (16px) will be used pretty much everywhere in the html.

For example:

If the default size is overridden to be 200px. Elements within that container will be rendered having font size of 200px. However if an element within that parent or container uses relative sizes, the browser will take the 200px as its bench mark.
