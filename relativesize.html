Relative size units in CSS3

Mar 17 2020

When it comes to setting the size of type in CSS, you have lots of options. It can either be a keywords like {xx-small, x-small, small, medium, large, x-large, and xx-large as well as relative keywords smaller and larger} or numerical values.

When using a value, we need to declare a unit of measure which itself has four options. The main difference between these units lays in how the browser determines the px value they translate into. Understanding this difference is the key to determining when to use each unit.

px

The pixel units (px) is by far the most popular and probably the most used unit. For any device, a reference pixel is defined based on resolution. The user agent scales px to match this reference pixel, and accordingly a size is set. Also, sizes are pretty consistent across all devices.
That being said, units defined in px are fixed. Element properties defined using px, will not be responsive by default media queries are needed to make them responsive because the user cannot change the font size in some browsers. For example, users with limited vision may wish to set the font size much larger than the size chosen by a web designer. Avoid using them for font sizes if you wish to create an inclusive design.

em

With em, sizes are always relative to the parent element. This would mean, if body is 10px, the font-size of h1 would be 20px and h2 would be 16px.

%

Percentage values are not units, but are used to define measurement properties. Percentage values are very similar to em units. The value set for a length or size unit using %, is relative to a property of either the same element, a parent element or that of the containing block.
If a parent has the font-size of 20px and the child has a font-size of 50%, it will come out to 10px. Just like em's percentage sizing is also relative. It also cascades in the same way, so the very problem described above of the list item and the paragraph item applies here as well.
But the compounding issue… For multi-level nested elements, em could potentially lead you to a compounding issue, where the size of nested elements, could get increasingly larger or smaller than intended. The more nesting the html has, the more it would keep on increasing or decreasing the font size.

rem

The rem unit is a computed value based on the size of the root element. And is best used when setting font-sizes. The root element, is one, which does not have a parent. In HTML, is the root element. In other words, it lets you specify a font size in a relative fashion without being affected by the size of the parent, thereby eliminating compounding.

Conclusion

The greatest power that rem units offer is not just that they give consistent sizing regardless of element inheritance. Rather, it’s that they give us a way to have user font size settings influence every single aspect of a site’s layout by using rem units where we used to use px units.
For this reason the primary purpose of using rem units should be to ensure that whatever default font size a user has their browser set to, the layout will adjust to accommodate the size of text within it.
A site can be designed initially focusing on the most common default browser font size of 16px.But by using relative size units, if a user increases their font size, the integrity of the layout will be preserved, and the text won’t get squashed up in a rigid space meant for smaller text.
